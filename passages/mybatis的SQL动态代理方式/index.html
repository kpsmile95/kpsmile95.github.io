  上文介绍了使用SqlSession的方式用mybatis对数据库进行操作。暴露了一些问题，在使用Mybatis基于SQLSession对象完成数据库操作时，参数的传递如果是多个参数需要我们封装成对象完成传递，但是就算只有两个我们也需要封装对象传递，就会比较麻烦，怎么办？

解决方案

  使用Sql动态代理方式完成数据库操作。

原理

  在没有使用mybatis时，我们会声明Dao层接口，然后自己创建并实现接口的实现类，在service层中调用接口的实现类完成数据库操作。
  而mybatis的sql动态代理方式，我们直接声明接口，但是Mybatis动态的产生接口的实现类，我们直接像以前一样在service层中调用动态生成的实现类完成数据库操作即可。

mapper动态代理开发的规范要求：

  1. 映射文件mapper.xml名字和接口名字一致。
  2. 映射文件的namespace要和接口的路径一致。
  3. 映射文件中的Statement的id要与接口的方法名一致。
  4. 映射文件的输入参数的类型：paramterType和接口的参数类型一致。
  5. 映射文件的输出结果 类型和接口的返回类型一致。
